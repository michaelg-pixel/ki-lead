Mailgun Consent Type Migration

Erweitere acceptance_type ENUM um 'mailgun_consent'

Was macht diese Migration?

Diese Migration erweitert das ENUM-Feld acceptance_type in der Tabelle av_contract_acceptances um den neuen Wert 'mailgun_consent'. Dies ermöglicht das separate Tracking der Mailgun-Zustimmung im Empfehlungsprogramm.

Wichtige Hinweise

  • Diese Migration ist non-destructive - keine Daten gehen verloren
  • Das ENUM wird von 3 auf 4 Werte erweitert
  • Bestehende Einträge bleiben unverändert
  • Neue Werte: registration, update, renewal, mailgun_consent
ALTER TABLE `av_contract_acceptances` MODIFY COLUMN `acceptance_type` ENUM('registration','update','renewal','mailgun_consent') NOT NULL DEFAULT 'registration';
